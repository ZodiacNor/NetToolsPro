### Wrappers med Windows-branch implementert

**net.py:**
- `ping_command(target, count, size, ttl, timeout_ms)` — Win: `ping -n -l -i -w`
- `ping_once_command(target, timeout_ms, size, ttl, df)` — med DF-støtte (Win: `-f`, Linux: `-M do`)
- `ping_once(target, timeout_ms)` — kjørbar variant
- `traceroute_command(target, max_hops, timeout_ms, resolve)` — Win: `tracert -d -h -w`
- `ipconfig_command()` — Win: `ipconfig /all`
- `netstat_command()` — Win: `netstat -ano`
- `arp_command(*args)` — Win + Linux
- `arp_table()` — parser + wrapper
- `arp_lookup(ip)` — parser + wrapper
- `default_gateway(adapter_ip)` — Win: route print -4 parser
- `default_gateway_output()` — rå output til historiske parsere

**shell.py:**
- `open_url(url)` — Win: `cmd /c start` / Linux: `xdg-open`
- `open_folder(path)` — Win: `explorer` / Linux: `xdg-open`

**scripting.py:**
- `script_extensions()` — Win: `{.ps1, .py, .bat, .cmd}` / Linux: `{.sh, .py}`
- `script_filetypes()` — tkinter filedialog-filtre
- `default_script_extension()` — Win: `.ps1` / Linux: `.sh`
- `build_run_command(path)` — bygger kjør-kommando

**detect.py:**
- `IS_WINDOWS`, `IS_LINUX`, `IS_MACOS`
- `SUBPROCESS_FLAGS` (CREATE_NO_WINDOW på Win)
- `is_admin()` — Win: `IsUserAnAdmin` / POSIX: `geteuid==0`

- `capabilities.suggest_setcap_command()` — fungerer, men bruker enkel $(which) (fase 10)
- `capabilities.relaunch_as_root()` — må implementeres med pkexec (fase 10)
- `parsers/linux.py` — `ip neigh` og default-route parser ferdig; `ip addr` + `ss -anop` gjenstår til Økt 3

## Beslutninger tatt underveis

1. **Combined fase 1+2 i samme omgang** — token-effektivt; Windows-impl skrevet direkte i stedet for stubs → flytting.
2. **Pakkestruktur (`platform_utils/`) valgt over én fil** — mer skalerbart.
3. **`ping_once_command()` utvidet med `df` og `ttl`-parametre** — for å støtte PingFrame's "Don't Fragment"-flagg cross-platform.
4. **`_get_gateway()` i DashboardFrame forenklet** — bruker nå `net.default_gateway()` direkte; fjernet dobbelt parsing.
5. **`_RUNNABLE_EXTS` er nå dynamisk** — settes via `_pu_scripting.script_extensions()` så den endrer seg basert på plattform.
6. **`SCRIPT_LAB_FILETYPES` bruker `_RUNNABLE_GLOB`** — bygges fra `_RUNNABLE_EXTS` så filedialog filtrerer riktig per plattform.
7. **Fase 7 deles i to** — `ip neigh` og `ip -4 route show default` er implementert først; `ip addr` og `ss -anop` utsettes til Økt 3 for å holde parserarbeidet avgrenset.
8. **`LinuxBackend` generator-stubs bruker `yield` etter `raise`** — nødvendig for at Python skal tolke metodene som generatorer (ikke vanlige funksjoner) selv om de alltid kaster. Uten `yield` ville kallet returnere umiddelbart i stedet for å kaste ved iterasjon.
9. **`_PS_DIAGNOSTICS` og `_SFC_RESULTS` ble modul-level konstanter i `system_backend.py`** — disse er ikke instans-state, og modul-level er idiomatisk for store streng-konstanter i Python.
10. **`nettools.py` ble redusert fra 8766 til 8638 linjer** (−128 linjer). Spec estimerte 8560–8620; vi fikk 8638 pga capability-guarded knapp-blokk er ~18 linjer lengre enn antatt — akseptabelt.
11. **`_do_backup` erstatter `_do_quick_backup`** — delegerer til `self._backend.export_services()`, håndterer exceptions/logging i framen.
12. **`_worker_sfc` fanger siste tag** — backend yielder summarylinje med `tag ∈ {success,warning,error}` på slutten; framen fanger og videresender til `_finish()`.

## Kjente problemer / caveats

1. **Windows-regresjonstest utført kun som statisk analyse + stubbed import.**
   Bengt må manuelt kjøre `build.bat` og verifisere alle 22 verktøy på Windows før vi går videre.
   Minimum regresjonsjekk: Ping, Traceroute, NetworkScanner, Interfaces, Connections, ARP, Dashboard (gateway), ScriptLab (kjør .ps1 og .py).

2. **DNS-frame (`nslookup`)** ble **ikke** wrappet — den er cross-platform som den er. Hvis vi vil kontroll, kan vi legge til `net.nslookup_command()` senere.

3. **PowerShell-, SFC- og DISM-kall** er UENDRET. De håndteres i fase 5 via `system_backend.py`.

4. **Interface-info-parsing (_with_ipconfig)** bruker fortsatt Windows-format. Linux `ip addr`-parser tas i Økt 3.

5. **Netstat-parsing** (ConnectionsFrame) antar Windows-kolonneformat. Linux `ss -anop`-parser tas i Økt 3.

6. **Tkinter er ikke installert på dev-maskinen.** Før fase 11-testing må Bengt kjøre:
   ```
   sudo apt install python3-tk python3-pil.imagetk
   ```

7. **Package import-test krevde stubbet `customtkinter`/`tkinter`.** Dette bekrefter syntaks men ikke runtime-oppførsel i GUI.
